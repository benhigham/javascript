# eslint-config namespaces its own blocks, keeps upstream preset names

Every flat-config block this package authors carries a `name` of `@benhigham/eslint-config/<slug>`, built through one `blockName` helper, so each block is addressable in `eslint --inspect-config` and the error messages that quote it — the comprehensive naming pass ADR-0008 deferred to #122. Upstream presets (`@eslint/js/recommended`, the `typescript-eslint/*` blocks, `@eslint-react/recommended-type-checked`, `config-prettier`) keep their own upstream names rather than being renamed under our namespace: they arrive through the spread that composes them and their names already read as meaningful provenance, so renaming would only erase it — the two blocks that adopt one this way (`base.js`, `react.js`) carry a comment so the missing namespace reads as intent. A `name` is tooling-visible only and never enters the resolved config, so this owes no changeset (CONTEXT.md, _Block_ / _Release coverage_).
